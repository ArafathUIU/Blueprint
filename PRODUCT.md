# Product

## Register

product

## Users

Product managers, startup founders, and developers who need to transform a raw product idea into a complete, structured product package — market research, user stories, wireframes, PRD, and development roadmap — without spending weeks across 6 different tools. Users are technical enough to value structured output (JSON, Markdown, SVGs) but want the speed of AI to collapse the research-to-spec pipeline into minutes.

## Product Purpose

Blueprint is an autonomous AI Product Manager agent. It takes a raw idea and produces a complete product package — research, stories, wireframes, PRD, roadmap — in one SSE-streamed workflow. Success means a user types an idea and gets engineering-ready output that's trustworthy (cited sources), structured (JSON-driven), and visually clear (Gantt charts, collapsible analysis, exportable Markdown).

## Brand Personality

**Precise, technical, minimal.** Think: a senior PM who knows their stuff and doesn't waste words. The product feels like a powerful tool you'd use in a terminal — dark, focused, monospace for agent output, red as the single accent of action. Output is the hero; chrome recedes. Confident and data-driven, never hand-wavy. No marketing fluff, no hype.

## Anti-references

- **Notion/Linear clones**: No generic SaaS look — soft blurple gradients, pill-shaped tags everywhere, Inter font as default, nested cards with drop shadows
- **Flashy AI demos**: No over-the-top particle effects, rainbow gradients, "revolutionary" copy, or holographic sci-fi aesthetics
- **Shadcn defaults**: While we use shadcn/ui primitives, avoid the stock shadcn look (gray-on-gray, identical radius, cookie-cutter card layouts)

## Design Principles

1. **Output over chrome** — The generated artifacts (research, stories, wireframes, PRD, roadmap) are the product. The UI serves them, not the other way around.
2. **Terminal as home** — The agent feels like a powerful CLI tool. Thinking streams live. Users see the agent work, building trust.
3. **Trust through transparency** — Citations, agent thinking blocks, viability scores. Users should verify, not just accept.
4. **One accent, one purpose** — Red (#DC2626) marks action, generation, and primary CTAs. Every other color serves a specific artifact type (amber=stories, cyan=roadmap, green=PRD, rose=wireframes).
5. **Dark-first, high contrast** — The workspace is dark (zinc-900/950), text is near-white. No gray-on-gray low-contrast elegance that sacrifices readability.

## Accessibility & Inclusion

- Target WCAG AA contrast (≥4.5:1 body text, ≥3:1 large text)
- Monospace font for agent output, system sans-serif for UI controls
- Reduced-motion alternatives for animations
- Keyboard-navigable terminal input and forms
